🔍 문제 원인 분석
─────────────────────────────────────────────────────
1. PumpyApp은 Expo가 아닌 "순수 React Native CLI" 앱입니다
2. 하지만 Expo의 virtual-metro-entry 파일을 찾으려 했습니다
3. 캐시가 손상되거나 잘못된 명령어로 실행되었습니다

✅ 해결한 것
─────────────────────────────────────────────────────
1. ✅ 기존 Node 프로세스 모두 중지
2. ✅ .expo 캐시 폴더 삭제
3. ✅ 포트 포워딩 재설정
4. ✅ react-native start --reset-cache로 Metro 시작
   (npm start가 아닌 올바른 명령어 사용)
5. ✅ 앱 다시 로드 명령 전송

  ⚠️ 중요: npm start vs react-native start
═══════════════════════════════════════════════════════

PumpyApp은 순수 React Native CLI 앱이므로:

❌ 틀린 방법: npm start
✅ 올바른 방법: npx react-native start

또는

✅ npx react-native start --reset-cache (캐시 문제시)

# 모든 Node 프로세스 중지
Stop-Process -Name node -Force

# 포트 설정
$env:LOCALAPPDATA\Android\Sdk\platform-tools\adb.exe reverse tcp:8081 tcp:8081

npx react-native start --reset-cache

# 앱 다시 로드
$env:LOCALAPPDATA\Android\Sdk\platform-tools\adb.exe shell input text "RR"

# 개발자 메뉴 열기
