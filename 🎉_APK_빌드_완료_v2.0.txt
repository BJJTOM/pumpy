방법 1: 자동 시작 (배치 파일)
   START_ALL.bat 실행
   → 백엔드와 웹 서버 모두 시작

방법 2: 수동 시작
   터미널 1 - 백엔드:
   cd C:\Users\guddn\Downloads\COCO\gym_api
   python manage.py runserver 0.0.0.0:8000

   터미널 2 - 웹:
   cd C:\Users\guddn\Downloads\COCO\gym_web
   npm run dev

   🚀 외부 인터넷 접속 (ngrok)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

같은 Wi-Fi가 아닌 곳에서 사용:

1. ngrok 설치 (이미 되어 있다면 skip)
   https://ngrok.com
   회원가입 및 Authtoken 설정

2. ngrok 실행:
   ngrok http 3000

3. ngrok URL 확인:
   예: https://abc123.ngrok-free.app

4. 앱에서 서버 설정:
   - 설정 → 서버 변경
   - ngrok URL 입력
   - 저장

5. 완료!
   어디서든 4G, 5G, 다른 Wi-Fi에서 접속 가능

Q: 앱에서 "Connection refused" 오류
A: 1. 백엔드 서버 실행 확인
   2. 웹 서버 실행 확인
   3. 서버 URL 정확한지 확인
   4. 방화벽 확인 (8000, 3000번 포트)

Q: 앱이 빈 화면만 표시
A: 1. 서버 설정 화면 나올 때까지 대기
   2. 서버 URL 입력
   3. 연결 테스트

Q: 서버 URL을 모르겠습니다
A: CMD에서 ipconfig 실행
   → IPv4 주소 확인
   → http://확인한IP:3000

Q: IP 주소가 계속 바뀝니다
A: ngrok 사용 권장
   → 고정 URL 제공
   → 외부 인터넷에서도 접속 가능

코드 수정 후 새 APK 빌드:

1. 코드 수정

2. Next.js 빌드:
   cd gym_web
   npm run build

3. Capacitor 동기화:
   npx cap sync android

4. APK 빌드:
   cd android
   $env:JAVA_HOME="C:\Program Files\Microsoft\jdk-21.0.8.9-hotspot"
   .\gradlew assembleDebug

5. APK 복사:
   Copy-Item "android\app\build\outputs\apk\debug\app-debug.apk" "..\Pumpy_v2.0_Updated.apk"

6. 새 APK 설치
