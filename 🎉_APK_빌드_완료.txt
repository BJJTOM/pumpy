   - 자신에게 파일 전송

   방법 C: Google Drive
   - Drive에 APK 업로드
   - 폰에서 다운로드

   방법 D: 이메일
   - 자신에게 APK 첨부하여 발송
   - 폰에서 열기


2️⃣ 알 수 없는 앱 설치 허용

   Android 8.0 이상:
   - 설정 → 앱 → 특수 앱 액세스
   - 알 수 없는 앱 설치
   - 파일 관리자/Chrome/Drive 등을 허용

   Android 7.0 이하:
   - 설정 → 보안
   - 알 수 없는 출처 허용

   - "열기" 또는 홈 화면에서 앱 실행

1️⃣ 서버 IP 주소 확인

   Windows:
   - 명령 프롬프트 실행 (cmd)
   - ipconfig 입력
   - "무선 LAN 어댑터 Wi-Fi"의 IPv4 주소 확인
   - 예: 192.168.0.15


2️⃣ 서버 실행

   백엔드 서버:
   cd C:\Users\guddn\Downloads\COCO\gym_api
   .\.venv\Scripts\python manage.py runserver 0.0.0.0:8000

   프론트엔드 서버:
   cd C:\Users\guddn\Downloads\COCO\gym_web
   npm run dev

   💡 두 서버 모두 실행되어 있어야 합니다!


3️⃣ 같은 Wi-Fi 연결

   - 컴퓨터와 폰이 같은 Wi-Fi에 연결되어 있어야 합니다
   - 공용 Wi-Fi의 경우 작동하지 않을 수 있습니다


4️⃣ 방화벽 설정

   Windows Defender 방화벽:

   - 고급 설정 → 인바운드 규칙 → 새 규칙
   - 포트 → TCP → 특정 로컬 포트: 3000, 8000
   - 연결 허용 → 완료

Q: 앱이 설치되지 않아요
A: - "알 수 없는 앱 설치 허용" 확인
   - 이미 설치된 앱이 있다면 먼저 삭제
   - 저장 공간 확인 (최소 100MB 필요)

Q: 앱이 실행되지 않아요
A: - 백엔드/프론트엔드 서버 실행 확인
   - 같은 Wi-Fi 연결 확인
   - 방화벽 설정 확인

Q: 흰 화면만 보여요
A: - 서버 실행 상태 확인
   - Chrome에서 http://당신의IP:3000 접속 테스트
   - 앱을 완전히 종료 후 다시 실행

Q: 데이터가 안 보여요
A: - capacitor.config.ts의 IP 주소 확인
   - 현재 설정: http://192.168.0.7:3000
   - 실제 IP와 다르면 수정 필요

현재 앱은 http://192.168.0.7:3000으로 설정되어 있습니다.

만약 IP가 다르다면:

1. 파일 열기:
   gym_web\capacitor.config.ts

2. 수정:
   server: {
     url: 'http://당신의실제IP:3000',
     cleartext: true
   }

3. 재빌드:
   cd gym_web
   npx cap sync android
   cd android
   .\gradlew.bat assembleDebug

4. 새 APK 위치:
   gym_web\android\app\build\outputs\apk\debug\app-debug.apk
