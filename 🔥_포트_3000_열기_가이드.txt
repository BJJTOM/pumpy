🎯 해결 방법 (5분이면 끝!)

══════════════════════════════════════════════════════════════

1단계: AWS 콘솔 접속
──────────────────────────────────────────────────────────────
   https://console.aws.amazon.com/

   로그인 후 EC2 서비스로 이동


2단계: 인스턴스 찾기
──────────────────────────────────────────────────────────────
   좌측 메뉴: "인스턴스" 클릭
   
   pumpy 서버 또는 IP "3.27.28.175" 찾기
   
   □ 체크박스 선택


3단계: 보안 그룹 열기
──────────────────────────────────────────────────────────────
   아래 탭에서 "보안" 탭 클릭
   
   "보안 그룹" 링크 클릭 (sg-xxxxx 형식)


4단계: 인바운드 규칙 편집
──────────────────────────────────────────────────────────────
   "인바운드 규칙" 탭 선택
   
   "인바운드 규칙 편집" 버튼 클릭


5단계: 규칙 추가
──────────────────────────────────────────────────────────────
   "규칙 추가" 버튼 클릭
   
   ┌─────────────────────────────────┐
   │ 유형: 사용자 지정 TCP           │
   │ 프로토콜: TCP                   │
   │ 포트 범위: 3000                 │
   │ 소스: 0.0.0.0/0                 │
   │ 설명: Next.js Web Frontend      │
   └─────────────────────────────────┘


6단계: 저장
──────────────────────────────────────────────────────────────
   "규칙 저장" 버튼 클릭
   
   완료! ✅

🆘 도움말

══════════════════════════════════════════════════════════════

Q: 보안 그룹을 찾을 수 없어요
A: EC2 → 인스턴스 → 인스턴스 선택 → 아래 "보안" 탭

Q: 규칙을 추가했는데도 안 돼요
A: 1) 규칙 저장 확인
   2) 브라우저 캐시 삭제 (Ctrl+Shift+Delete)
   3) 시크릿 모드로 재접속

Q: 0.0.0.0/0이 안전한가요?
A: 웹사이트는 공개되어야 하므로 안전합니다.
   SSH(22)만 본인 IP로 제한하세요.
