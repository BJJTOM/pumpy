═══════════════════════════════════════════════════════════════
         방법 1: 자동 스크립트 실행 (권장 ⭐)
═══════════════════════════════════════════════════════════════

1️⃣ AWS에 SSH 접속:
   ssh -i your-key.pem ubuntu@3.27.28.175

2️⃣ 스크립트 다운로드:
   wget https://raw.githubusercontent.com/YOUR_REPO/update_django_admin.sh
   
   또는 직접 생성:
   nano update_django_admin.sh
   (아래 스크립트 내용 복사/붙여넣기)

3️⃣ 실행 권한 부여:
   chmod +x update_django_admin.sh

4️⃣ 스크립트 실행:
   ./update_django_admin.sh

5️⃣ 완료! 
   브라우저에서 http://3.27.28.175:8000/admin/ 접속
   Ctrl+Shift+R 로 강력 새로고침!

═══════════════════════════════════════════════════════════════
         방법 2: 수동으로 한 줄씩 실행
═══════════════════════════════════════════════════════════════

1️⃣ SSH 접속:
   ssh -i your-key.pem ubuntu@3.27.28.175

2️⃣ 프로젝트 디렉토리로 이동:
   cd ~/gym_api
   # 또는 find / -name "manage.py" 2>/dev/null | grep gym

3️⃣ Static 디렉토리 생성:
   mkdir -p members/static/admin/css

4️⃣ CSS 파일 생성:
   nano members/static/admin/css/custom_admin.css

5️⃣ CSS 내용 붙여넣기:
   (아래 CSS 코드 전체 복사/붙여넣기)
   Ctrl+O (저장), Enter, Ctrl+X (종료)

6️⃣ Static 파일 수집:
   python3 manage.py collectstatic --noinput

7️⃣ 서버 재시작:
   sudo systemctl restart gunicorn
   sudo systemctl restart nginx

8️⃣ 완료!
   브라우저에서 http://3.27.28.175:8000/admin/ 접속
   Ctrl+Shift+R 로 강력 새로고침!

═══════════════════════════════════════════════════════════════
         방법 3: AWS Console (Session Manager)
═══════════════════════════════════════════════════════════════

1️⃣ AWS Console 접속:
   https://console.aws.amazon.com/

2️⃣ EC2 → 인스턴스 → 3.27.28.175 선택

3️⃣ "연결" 버튼 → "Session Manager" 탭

4️⃣ "연결" 클릭 (브라우저에서 터미널 열림)

5️⃣ 위의 "방법 2" 명령어들 실행

6️⃣ 완료!

         🔍 문제 해결
═══════════════════════════════════════════════════════════════

❓ 디자인이 변경되지 않는다면:

1. 브라우저 캐시 강력 삭제:
   Ctrl+Shift+Delete → 캐시 삭제 → 재접속

2. Static 파일 경로 확인:
   python3 manage.py findstatic admin/css/custom_admin.css

3. 파일 권한 확인:
   ls -la members/static/admin/css/custom_admin.css
   chmod 644 members/static/admin/css/custom_admin.css

4. 서버 로그 확인:
   sudo journalctl -u gunicorn -n 50
   sudo tail -f /var/log/nginx/error.log

5. Gunicorn/Nginx 상태:
   sudo systemctl status gunicorn
   sudo systemctl status nginx
